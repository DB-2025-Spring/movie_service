프론트엔드:
• 개인정보 수집 및 이용 동의 항목을 체크박스로 제공한다.
    ◦ 체크하지 않을 경우 예매 진행 불가 및 안내 문구 표시
• 입력 필드 구성:
    ◦ 법정생년월일 (8자리, 숫자만 입력)
    ◦ 휴대전화번호 (통신사 선택 가능, 인증번호 요청 버튼 제공)
    ◦ 비밀번호 및 비밀번호 확인란
• "예매하기" 버튼 클릭 시 필수 항목 누락 여부를 검증하고 안내 메시지 출력
• 모든 입력값은 유효성 검사 후 다음 단계로 이동

백엔드:
• 법정생년월일, 휴대폰번호, 비밀번호를 바탕으로 비회원 예매자 정보 생성 또는 확인
• 예매 확정 시 해당 정보와 함께 티켓 발권 정보 저장
• 예매 내역 조회 시 해당 정보 기반으로 검증/조회

프론트엔드:
(비회원으로 결제시): 결제수단을 선택하는 라디오 박스를 랜더링한다.
(회원으로 결제시): 할인쿠폰 / 포인트 사용/ 결제수단을 선택하는 라디오 박스를 랜더링한다.
관람등급 미만의 고객이 결제 버튼을 누르면 경고문과 함께 메인페이지로 이동 버튼을 제공한다.

실시간으로 결제할 금액 / 총 할인금액/ 남은 결제금액을 랜더링한다.

사용자가 이들을 모두 작성을 완료 할 경우, 결제하기 버튼이 활성화 된다. 결제하기 버튼을 클릭할 시, 결제창을 랜더링하고, 카드번호를 입력받고 백엔드에게 데이터를 전달한다. 백엔드로 부터 응답을 결과로 나의 예매내역으로 리다이렉트/결제창을 다시 랜더링 한다.

백엔드:
(임시)프론트엔드로 부터 전달받은 총 결제금액과 결제 방법을 바탕으로, 결제 트랜젝션을 시작한다. 결제 트랜젝션이 성공적으로 commit되었을 경우, 좌석선택 트랜젝션을 commit 한다. 결제 트랜젝션이 실패했을 경우, 결제 트랜젝션을 rollback한다. 결제 트랜젝션 결과를 프론트 엔드에게 전달한다.
